## API Overview

- **Style:** REST
- **Base URL:** https://api.mossbridge-analytics.example.com/v1
- **Current version:** v1
- **Data format:** JSON
- **Authentication:** API key
- **Sandbox available:** Yes — https://sandbox-api.mossbridge-analytics.example.com/

### What Developers Can Do

- Ingest facility telemetry from approved edge gateways
- Query historical monitoring data by site, metric, and date range
- Create and retrieve sustainability reporting exports (PDF/CSV)
- Subscribe to alerts for threshold breaches via webhooks

## Authentication & Access

### Obtaining Credentials
Register a developer account, create a project in the developer portal, then generate API keys from the project settings page.

### Key Types
- **Test keys:** Prefixed with `test_` — sandbox environment only
- **Live keys:** Prefixed with `live_` — production access

### Rate Limits

| Tier        | Limit             | Burst |
|-------------|-------------------|-------|
| Starter     | 120 req/min       | 180   |
| Growth      | 800 req/min       | 1200  |
| Enterprise  | Custom contract   | Custom|

Rate limit headers: `X-RateLimit-Limit`, `X-RateLimit-Remaining`, `X-RateLimit-Reset`

## Webhooks

- **Available:** Yes
- **Setup:** https://docs.mossbridge-analytics.example.com/webhooks/
- **Signature verification:** HMAC-SHA256 with shared secret
- **Retry policy:** Exponential backoff, up to 8 retries over 24h

---

## Error Handling

Standard HTTP status codes apply. Error responses include:

```json
{
  "error": {
    "code": "invalid_parameter",
    "message": "Human-readable description",
    "param": "field_name",
    "doc_url": "https://docs.mossbridge-analytics.example.com/errors/invalid_parameter"
  }
}
```

Common codes: `400` (bad request), `401` (unauthorised), `403` (forbidden), `404` (not found), `429` (rate limited), `500` (server error)
